Задача: доработать UI/UX блока выбора периода для WB API-аналитики.

Нужно сделать отдельный логический блок “Выберите период”, который отображается, если у пользователя добавлен WB API-ключ.

Цель блока:
пользователь выбирает период, за который нужно получить детализированный отчёт WB через API.

Структура блока:

1. Заголовок:
   “Выберите период”

2. Короткое описание:
   “Получите детализированный отчёт WB через API за нужный период.”

3. Быстрые пресеты периода:
- 7 дней
- 14 дней
- 30 дней
- Текущий месяц
- Прошлый месяц
- Свой период

4. Поле выбора периода:
   использовать один range-picker, а не два отдельных инпута.

Формат поля:

[ 01.04.2026 — 28.04.2026 ▼ ]

По клику открывается календарь с выбором диапазона дат.

5. Основная кнопка:
   “Получить отчёт WB”

6. Подсказка под периодом:
   “Будет получен отчёт WB за выбранный период.”

После успешного получения отчёта показывать состояние:

“Отчёт получен через API”
“Период аналитики: DD.MM.YYYY – DD.MM.YYYY”
“Строк в отчёте: N”

Кнопки:
[Обновить] [Сбросить]

Логика пресетов:

- “7 дней” — последние 7 календарных дней.
- “14 дней” — последние 14 календарных дней.
- “30 дней” — последние 30 календарных дней.
- “Текущий месяц” — с 1 числа текущего месяца по сегодня.
- “Прошлый месяц” — полный предыдущий календарный месяц.
- “Свой период” — пользователь выбирает диапазон вручную через range-picker.

Валидация:

1. Если период не выбран — кнопка “Получить отчёт WB” disabled.
2. Если период слишком длинный — показать ошибку:
   “Выберите период не больше 31 дня.”
3. Если дата начала больше даты конца — показать ошибку:
   “Дата начала не может быть позже даты окончания.”
4. Пока отчёт загружается — кнопка disabled, текст:
   “Получаем отчёт...”

Рекомендации по UI:

- Блок должен выглядеть как отдельная карточка.
- Пресеты оформить как компактные pills/buttons.
- Активный пресет визуально выделять.
- Если пользователь вручную меняет даты, активным становится “Свой период”.
- Range-picker должен быть основной точкой выбора дат.
- Не делать два отдельных поля “Дата начала” и “Дата конца” как основной UX.

Финальная структура карточки:

Выберите период

Получите детализированный отчёт WB через API за нужный период.

[7 дней] [14 дней] [30 дней] [Текущий месяц] [Прошлый месяц] [Свой период]

Период:
[ 01.04.2026 — 28.04.2026 ▼ ]

Будет получен отчёт WB за выбранный период.

[Получить отчёт WB]
